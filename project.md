# 基于大模型的测试场景生成与智能测试工具

## 一、项目简介

1. 随着 Web Agent 快速发展，行业对其在复杂 Web 应用场景下的测试可用性提出了更高要求。
2. 当前现有 Web Agent 在 WebArena 等测试基准中任务失败率较高，核心原因分为三点：
   （1）Web 应用业务逻辑复杂，且具备极强的领域专属特性；
   （2）大模型存在幻觉问题，同时缺少待测软件的领域专业知识，容易输出不符合真实业务的错误信息；
   （3）完成操作后，难以精准判定当前测试任务是否执行成功。
3. 软件配套文档（如官方用户手册）包含完整业务知识，可辅助智能体理解系统功能、校验操作执行结果。
4. 本次课程实践目标：设计一套基于大模型的测试场景自动生成、自主执行一体化测试工具，核心实现两大能力：
   （1）自动解析提取用户手册内业务知识，生成覆盖完整、粒度适中、可直接运行的标准化测试场景；
   （2）搭建大模型智能体，自动运行生成的测试场景，完成目标Web应用全量功能测试。

## 二、核心任务

### 任务一：基于用户手册的测试场景自动生成

1. 实现目标
   依托大模型检索增强生成技术（RAG），从软件用户手册中抽取业务知识，自动化生成测试场景。
2. 功能要求
   （1）可从用户手册中提取软件全部核心功能点；
   （2）针对每一个功能点，输出结构化标准测试场景；
   （3）支持功能点、生成测试场景的可视化展示。
3. 测试场景标准结构
   整体格式：`[[操作步骤]+ [预期状态]?]+`
   - 操作步骤：完成该功能所需的分步操作流程；
   - 预期状态：功能执行完成后的标准结果（测试预言），包含判定功能是否执行成功的关键特征。

### 任务二：测试场景驱动的智能测试智能体

1. 实现目标
   搭建基于大模型的Web端测试智能体，自动执行任务一生成的全部测试场景。
2. 架构必备模块（可自行拓展新增模块）
   （1）规划模块：读取测试场景，自主拆解、生成完整执行计划；
   （2）记忆模块：存储、维护测试执行所需全部上下文信息；
   （3）执行模块：与浏览器页面交互，自动落地规划内的各项操作；
   （4）验证模块：结合完整测试执行轨迹，校验场景是否执行成功、功能运行是否正常。

## 三、评分标准

### （一）基础功能档

1. 任务一达标要求：可识别用户手册中的核心功能点，并按照规范格式生成对应测试场景；
2. 任务二达标要求：智能体能够自动运行任务一中生成的简单测试场景，可校验操作完整度与功能运行正确性。

### （二）提升创新档

#### 任务一评价维度

1. 测试场景正确性与覆盖完整性
   - 保障从文档检索获取的业务信息真实有效，规避大模型幻觉带来的场景错误；
   - 完整覆盖文档记载的全部功能，无功能遗漏。
2. 测试场景粒度合理性与可执行性
   - 功能拆分粒度适中：既不会拆分过细导致无法组合成完整业务流程，也不会划分过粗，无法指导具体页面操作；
   - 输出的用户故事可直接被现有Web测试智能体识别并执行。

#### 任务二评价维度

1. 场景执行通过率与运行稳定性
   - 提升智能体执行准确率，可正常通过简单、中等难度测试场景，支持部分复杂场景执行；可优化整体执行效率。
2. 支持测试场景变异检测，可识别多种典型应用缺陷：操作执行异常、页面布局错乱、文本语义错误等。
3. 测试结果自动校验：依托完整执行轨迹准确区分执行成功/失败状态，针对失败用例输出清晰的失败原因。

> 补充鼓励方向：自主设计创新技术方案、自定义测试评估策略、设计专属评估指标等均可加分。
